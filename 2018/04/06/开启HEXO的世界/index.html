发表于 2018-04-06 | 更新于: 2018-04-06 | 分类于 Hexo
字数统计: 817 字 | 阅读时长 ≈ 3 分钟

心血来潮

刷知乎的时候突然看到有个用户的个人Blog，界面设计得很好看，也没有广告，也不是什么CSDN之类的，感觉很好奇，就研究了一下，发现底部有个入口HEXO，初步研究之后发现这东东可以实现我多年的想建立独立Blog的想法啊。为什么这么说呢，首先只要有个github账号就可以。不需要购买云主机来搭建，虽然说空间不大，但是个人使用还是够用的。另外就是如果舍不得买域名，可以直接访问github的地址，虽然国内慢了点，但是省事啊。最后就是维护简单，几条命令搭建好了之后就可以写文章了。即便什么都不懂的小白也能操作。好了废话不多说，接下来就开干吧。

找教程的过程中先要膜拜一下祖师爷 Linus和大神tommy351,前者写出了linux和git，后者开发了hexo。

  • 写得比较好的教程
    GitHub+Hexo 搭建个人网站详细教程，写得还不错。
  • Hexo的官方文档
    Hexo文档，（找到的时候已经参考上面教程搭建的差不多了，说实话官方文档我觉得写得更好）
  • git使用教程
    廖雪峰的git教程，又是一位大神，非常通俗易懂，有图有视频。

踩的坑

  • node的版本
    我的linux系统是Ubuntu 1604，之前鼓捣的时候把apt的源搞成网易的了，后来发现通过网易的源安装的node版本太低了，安装最新的hexo各种错误。然后想着切换到阿里的源会不会新一点，结果切过去之后仍然是比较老的版本。郁闷。度娘了一下怎么单独升级node。发现挺简单的,两条命令搞定。
    npm install -g n
    n stable
  • Hexo init失败
    安装过程中，出现初始化失败的问题。好像没什么好办法，我是重新删除掉node和npm，重新按照流程又搞了一遍就好了。
  • 部署的时候网页主题显示不出来
    已经把所有文件push到github上了，但是通过github的地址访问总是不显示主题，查了一下文档，_config.yml的root和url字段配置不对。url就填写GitHub的仓库路径。root目录应该是root: /子目录/这样子。传送门：puliu的回答。
  • github pages配置
    在github代码仓库的settings界面里可以修改一下github pages设置为master分支。针对github pages官网是这么介绍的，提供的资源也是有限制的。

    GitHub Pages is designed to host your personal, organization, or project pages from a GitHub repository.

    GitHub Pages sites are subject to the following usage limits:
    GitHub Pages source repositories have a recommended limit of 1GB .
    Published GitHub Pages sites may be no larger than 1 GB.
    GitHub Pages sites have a soft bandwidth limit of 100GB per month.
    GitHub Pages sites have a soft limit of 10 builds per hour.
